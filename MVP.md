# MVP 方案 — AI 对话说服游戏(working title:《嘴遁》/ TalkOut)

> 定位:面向**刷短视频的泛玩家**,做"一局 1–3 分钟、每局都不一样、玩家忍不住截图发出去"的 AI 对话小游戏。
> 北极星指标:**单局产出"可分享瞬间"的概率**(可量化为:分享卡导出率 / 自然传播带来的新用户占比)。

---

## 1. 核心概念(One-liner)

> 你只有几轮对话,要用一张嘴说服一个**有性格、有底线、有隐藏弱点**的 AI 角色,让它做一件它本来死活不愿意做的事。
> 说服成功 → 生成一张"战报卡"可一键分享;踩到它的红线 → 当场翻车(翻车往往最好笑、最可传播)。

这是 **Suck Up! 的玩法内核** + **Wordle 的每日同题对比** + **战报卡的社交分发**。

---

## 2. 核心循环(Core Loop,单局 ≤ 3 分钟)

```
进入场景  →  看到 AI 角色 + 你的目标(它不愿意做的事)
   ↓
对话回合(上限 6–8 轮):你说一句 → AI 即时回应(带表情/语气)
   ↓
隐藏的"说服度"在涨/跌;说错话可能踩红线 = 直接翻车
   ↓
结局:说服成功 / 回合耗尽 / 踩红线翻车
   ↓
生成「战报卡」(金句 + 结局 + 用了几轮)→ 一键分享 / 再来一局
```

**关键设计:每局随机** 角色性格、目标、隐藏弱点、红线 → 保证不可预测、可重复玩、可比较。

---

## 3. 让"分享瞬间"可控发生的机制(这是项目成败核心)

| 机制 | 怎么制造分享欲 |
|---|---|
| **隐藏弱点(soft spot)** | 每个 AI 有个只能靠试探挖到的弱点(怕老婆/贪小便宜/中二/想退休)。挖到 = "啊哈"时刻 |
| **红线翻车(hard line)** | 踩线时 AI 会有夸张、出戏或冷场反应 —— 失败比成功更好笑、更想发出去 |
| **AI 即兴金句** | 让 AI 偶尔抛出意料之外的吐槽/反问。不可预测性是卖点不是 bug |
| **每日同题(Daily)** | 所有人今天面对同一个 AI + 同一目标 → 比谁用的话术更骚、轮次更少,天然话题 |
| **战报卡** | 自动截取本局最炸的 1–2 句对话 + 结局印章,做成竖版卡片(适配短视频/朋友圈) |
| **排行榜** | "最少轮次通关""最骚话术"(由 AI 给本局打一个风格分) |

## 4. 技术选型(Web 优先,空目录起步)

```
[ 浏览器 SPA ]  ──HTTPS──▶  [ 轻后端 / Serverless ]  ──API──▶  [ LLM(NPC 大脑)]
    │                              │
  战报卡渲染                   隐藏 API Key + 判定胜负 + 防刷
```

- **前端**:Next.js(App Router)或纯 Vite + React。单页、移动端竖屏优先。战报卡用 `html-to-image` / `satori` 导出 PNG。
- **后端**:Next.js Route Handler 或一个 serverless function 即可。职责:① 代理 LLM 调用、藏 key;② 用结构化输出做胜负判定;③ 限流防刷。
- **NPC 大脑**:**Claude Haiku 4.5**(`claude-haiku-4-5-20251001`)—— 便宜、快(对话游戏延迟必须低),性格扮演足够。每轮一次调用。
  - 用 **tool use / 结构化输出**,让模型每轮同时返回:
    ```json
    {
      "reply": "角色台词",
      "emotion": "wary | amused | annoyed | won_over",
      "persuasion_delta": -20..+20,   // 本轮说服度变化
      "redline_hit": false             // 是否踩红线 → 直接翻车
    }
    ```
  - 这样**一次调用既扮演角色又当裁判**,省一次延迟和成本。
- **角色生成**:每个角色 = 一段 system prompt 模板(性格 + 目标 + 隐藏弱点 + 红线 + 说服阈值)。MVP 先**手写 8–12 个**精修角色,别上来就全自动生成(质量不可控)。
- **状态**:说服度(0–100,隐藏)、回合数。无需账号即可玩;排行榜/Daily 再加轻量存储(Upstash Redis / SQLite)。

> 模型成本估算:单局 ~6–8 次 Haiku 调用、每次输入数百 token,**单局成本 < 1 美分**,可承受免费试玩冷启动。

## 5. MVP 范围(明确做 / 不做)

### ✅ 做(第一个可玩版本)
- 1 个核心场景框架 + **手写 8 个角色**(各自性格/目标/弱点/红线)
- 完整核心循环:对话 → 说服度 → 三种结局
- **战报卡导出**(没有这个就没有传播,必做)
- 移动端竖屏 UI、打字机式逐字出字、简单表情/语气提示
- 基础防刷限流

### ❌ 暂不做(MVP 后再说)
- 账号系统 / 登录
- 排行榜 & 每日同题(第二迭代,验证留存后再上)
- 语音 / 对口型(Whispers 那套,贵且慢,先纯文字)
- 自定义角色 / UGC
- 本地推理、付费、内购
- 复杂美术、3D

---

## 6. 里程碑(建议节奏)

| 阶段 | 产出 | 验证目标 |
|---|---|---|
| **M0 原型(3–5 天)** | 1 个角色跑通对话→判定→结局的最小闭环(可丑) | 玩法到底好不好玩、AI 判定靠不靠谱 |
| **M1 可玩 Demo(2 周)** | 8 角色 + 战报卡 + 移动端 UI | 朋友能玩、玩完会不会想发出去 |
| **M2 软上线(再 2 周)** | 排行榜 + 每日同题 + 埋点 | 自然传播跑不跑得通、次日留存 |

---

## 7. 成功 / 失败判定指标

- **分享卡导出率**(玩完一局有多少 % 导出/分享) ← 最重要,目标 > 15%
- 单局时长落在 1–3 分钟
- 人均局数(可重复玩性) ← 目标 > 3 局/次
- 自然新增占比(靠分享带来的新用户)
- 反向红灯:有人骂"这就是 AI 凑数(gameslop)" → 说明角色/金句质量不够,回去精修 prompt

---

## 8. 最大风险 & 对策

| 风险 | 对策 |
|---|---|
| AI 判定胡来(该赢没赢/该输没输) | 结构化输出 + 说服阈值显式化 + 大量人工调 prompt;M0 重点测这个 |
| 角色无聊、没金句 = 没人分享 | MVP 不靠自动生成,**手工精修每个角色的弱点与红线**,把"好笑"调出来 |
| 延迟高,对话卡顿 | 选 Haiku、流式输出、UI 上用打字机动画掩盖等待 |
| 玩家想越狱/刷屏脏话 | system prompt 设角色边界 + 后端限流 + 红线机制本身就吸收了"乱说" |
| 成本失控 | 限流 + 单局轮次上限 + Haiku 已极便宜 |

---

## 9. 下一步(开工第一刀)

先做 **M0**:1 个角色(建议"死活不让你进门的 AI 保安")+ 后端一个 route + 结构化输出判定。
跑通后只问一个问题:**"我玩完想不想再来一局 / 想不想发给朋友?"** —— 是,就继续;不是,就改角色而不是改技术。
